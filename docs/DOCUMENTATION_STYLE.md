# Стиль документации репозитория

Материалы вроде `README.md`, `docs/*.md`, `AGENTS.md`, `.cursorrules`, `k8s/README.md` читают инженеры и аналитики вне контекста чата. Текст должен быть самодостаточным и полезным для стороннего читателя.

## Принципы

1. **Конструктивное описание** — опирайтесь на то, что система делает и откуда берутся данные («цепочка источников…», «команда запускает…»). Не раздувайте документ перечислением того, чего в поле или файле «нет», если это не жёсткое требование безопасности или совместимости.
2. **Аудитория** — пишите для человека, который открыл репозиторий впервые: достаточно доменного контекста без отсылок к обсуждению в мессенджере или к истории правок в чате.
3. **Без цитирования переписки** — не вставляйте дословные реплики из чата и не оформляйте абзацы в виде «вам сказали сделать X». Итог читается как единая справочная статья.
4. **Согласованность терминов** — один и тот же термин (например MSGTEXT, LOGTEXT, JID) везде в одном смысле; при изменении поведения в коде обновляйте связанные разделы в README, `AGENTS.md`, `.cursorrules` и соответствующих `docs/`.
