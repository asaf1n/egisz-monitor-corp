# EGISZ Monitor Corp

**EGISZ Monitor Corp** — специализированный ETL-сервис для мониторинга интеграции Медицинских Информационных Систем (МИС) с реестрами ЕГИСЗ (в частности, РЭМД). Сервис выполняет извлечение данных из очередей Firebird, парсинг SOAP-ответов, нормализацию и загрузку в хранилище PostgreSQL для последующей визуализации.

### Цели и задачи сервиса
* Автоматизированный сбор логов взаимодействия с ЕГИЗС из СУБД Firebird.
* Десериализация SOAP-сообщений для извлечения статусов регистрации и идентификаторов документов.
* Формирование аналитических витрин данных в PostgreSQL для контроля качества интеграции.
* Обеспечение техподдержки инструментом для анализа ошибок парсинга и передачи данных.

### Стек технологий
* **Ядро:** Python 3.10+.
* **СУБД Источник:** Firebird (через `firebird-driver`).
* **Хранилище (DWH):** PostgreSQL 15/16.
* **Оркестрация:** Apache Airflow (исполнение DAG по расписанию).
* **Визуализация:** Metabase.
* **Инфраструктура:** Docker, Kubernetes (namespace `egisz-corp`).

### Техническая реализация и логика парсинга

#### Инкрементальная загрузка
Синхронизация выполняется методом инкрементальной выборки. В качестве контрольной точки (watermark) используется поле `EXCHANGELOG.LOGID`, значение которого сохраняется в таблице `etl_state`. Это исключает пропуски при обновлении строк в источнике.

#### Сопоставление данных (Mappings)
Для формирования корректной записи в витрине данных `fact_egisz_transactions` используются следующие правила:

**1. Идентификация документа и корреляция:**
* **Связь с ответом:** Тег `<relatesToMessage>` из SOAP-ответа в `EXCHANGELOG.LOGTEXT` сохраняется как основной ключ `relates_to_id`.
* **Внутренний ID:** Идентификатор документа `DOCUMENTID` (также передается как `localUid`) берется из таблицы `EGISZ_MESSAGES`.

**2. Идентификация юридического лица (Клиники):**
* **Через URL:** Значение `JID` извлекается из строки хоста в `EXCHANGELOG.LOGTEXT` по регулярному выражению `gost-([a-zA-Z0-9_-]+)\.infoclinica\.lan`.
* **Через OID:** Тег `<organization>` из XML сопоставляется с полем `EGISZ_LICENSES.MO_UID` для получения `JID`.
* **Через домен:** Подстрока из `EGISZ_MESSAGES.REPLYTO` сопоставляется с `EGISZ_LICENSES.MO_DOMEN`.
* **Наименование:** По полученному `JID` из таблицы `JPERSONS` извлекается `JNAME` (наименование клиники).

**3. Классификация типов документов:**
* Код типа документа (KIND) извлекается из тега `<kind>` в XML или из `EGISZ_LICENSES.KIND`.
* Наименование типа документа определяется по справочнику НСИ 1.2.643.5.1.13.13.11.1520.

### Инфраструктура и доступ

#### Основные компоненты и порты
| Сервис | Адрес в K8s / Локально | Назначение |
| :--- | :--- | :--- |
| **PostgreSQL** | `postgres.egisz-corp:5432` / порт 5433 | Хранение витрин данных. |
| **Metabase** | `metabase.egisz-corp:3000` / порт 3001 | Визуализация дашбордов. |
| **Config UI** | `corp-web.egisz-corp:8080` / порт 8080 | Редактор YAML-конфигурации. |

#### Конфигурация
Настройки подключений хранятся в файле `config/egisz_corp.yaml` или передаются через переменные окружения (`EGISZ_CORP_POSTGRES_HOST` и др.).
